# 共通プロンプト・指示書

> AI エージェント（Claude Code 等）が各スキルを実行する際に共通で従うべき指示。

## 基本原則

1. **既存コードの尊重**: 変更前に必ず既存コードを読み、スタイル・規約を踏襲する
2. **最小限の変更**: 依頼された範囲のみ変更する。スコープ外の改善は提案に留める
3. **安全性**: 機密情報の漏洩、破壊的操作、セキュリティ脆弱性の導入を避ける

## コミュニケーション

- 応答は日本語
- コード・コミットメッセージ・ブランチ名は英語
- 端的に要点のみ伝える。冗長な説明は不要

## コーディング規約

- プロジェクトの既存規約に従う（新しい規約を勝手に導入しない）
- コミットメッセージは Conventional Commits 形式（`feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`）
- 機能変更とリファクタリングはコミットを分ける

## 推測ルール

不明点があっても推測で進めてよい。ただし:

- 該当コード内に `// ASSUMPTION:` コメントで推測箇所を明記する
- `ASSUMPTIONS.md` に推測の記録を追記する

## 品質チェック

- 変更後はリンター・テストを実行して確認する
- テストが存在しない場合はその旨を報告する

## 作業フロー（共通）

```
1. 対象コードを読んで理解する
2. 変更計画を立てる
3. 実装する
4. リンター・テストで検証する
5. 結果を報告する
```
